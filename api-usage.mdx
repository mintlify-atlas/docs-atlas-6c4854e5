When you call DeepSeek V4 through the API, you inject the control instruction by appending it to the first user message in your messages list. This page shows the helper functions from the reference implementation and explains how multi-turn conversations work once the instruction is in place.

The instructions apply to both `deepseek-v4-flash` and `deepseek-v4-pro`.

## Instruction marker constants

Define the two instruction strings as constants. Copy these exactly — the model was trained with this specific injection format.

```python
INNER_OS_MARKER = (
    "\n\n【角色沉浸要求】在你的思考过程（<think>标签内）中，请遵守以下规则：\n"
    "1. 请以角色第一人称进行内心独白，用括号包裹内心活动，例如"（心想：……）"或"(内心OS：……)"\n"
    "2. 用第一人称描写角色的内心感受，例如"我心想""我觉得""我暗自"等\n"
    "3. 思考内容应沉浸在角色中，通过内心独白分析剧情和规划回复"
)
NO_INNER_OS_MARKER = (
    "\n\n【思维模式要求】在你的思考过程（<think>标签内）中，请遵守以下规则：\n"
    "1. 禁止使用圆括号包裹内心独白，例如"（心想：……）"或"(内心OS：……)"，所有分析内容直接陈述即可\n"
    "2. 禁止以角色第一人称描写内心活动，例如"我心想""我觉得""我暗自"等，请用分析性语言替代\n"
    "3. 思考内容应聚焦于剧情走向分析和回复内容规划，不要在思考中进行角色扮演式的内心戏表演"
)
```

## The `build_messages` helper

Use `build_messages` to construct the initial messages list. It handles marker injection automatically based on the `mode` argument.

```python
def build_messages(system_prompt, user_first_message, mode="default"):
    if mode == "inner_os":
        user_first_message += INNER_OS_MARKER
    elif mode == "no_inner_os":
        user_first_message += NO_INNER_OS_MARKER
    return [
        {"role": "system", "content": system_prompt},
        {"role": "user",   "content": user_first_message},
    ]
```

### Parameters

<ParamField body="system_prompt" type="string" required>
  The system prompt that defines the character's personality, background, and rules. For example: `"你是一个傲娇的女高中生..."`.
</ParamField>

<ParamField body="user_first_message" type="string" required>
  The first user message that opens the roleplay scene. The marker is appended to this string when a mode other than `"default"` is specified.
</ParamField>

<ParamField body="mode" type="string" default="default">
  Controls which instruction is injected. Accepted values:

  - `"default"` — no instruction appended; the model chooses its thinking style automatically.
  - `"inner_os"` — appends `INNER_OS_MARKER`; activates first-person character immersion in the `<think>` block.
  - `"no_inner_os"` — appends `NO_INNER_OS_MARKER`; suppresses inner-monologue and enforces analytical thinking.
</ParamField>

<Note>
  Put the instruction in the first user message, not in the system prompt. The model was trained with the instruction injected at the end of the first user turn. Placing it in the system prompt produces less consistent results.
</Note>

## Multi-turn conversation example

Call `build_messages` once for the first turn, then append subsequent messages normally. The instruction persists automatically.

```python
# Turn 1: marker is injected into the first user message
messages = build_messages(
    "你是一个傲娇的女高中生...",
    "「我走进教室」"早上好。"",
    mode="inner_os"
)
response = client.chat(messages)

# Turn 2 onward: append as usual, no further handling needed
messages.append({"role": "assistant", "content": response})
messages.append({"role": "user", "content": "「我在她旁边坐下」"今天心情不好吗？""})
response = client.chat(messages)  # the marker from turn 1 is still in history
```

<Tip>
  The instruction stays active for the entire conversation because the model receives the full message history on every call. The marker you added to the first user message is always present in the context — you do not need to re-inject it on subsequent turns.
</Tip>

## Effect on responses

The instruction only affects the content inside the `<think>` block, not the final visible reply. However, thinking style does influence output quality:

- **`inner_os` mode** — more emotionally authentic replies due to first-person immersive reasoning.
- **`no_inner_os` mode** — more structurally consistent replies due to analytical reasoning.

See [Instructions reference](/instructions-reference) for the raw instruction strings and guidance on choosing between modes.
